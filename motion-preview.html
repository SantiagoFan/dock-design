第 1 段 · 滚动这块容器，每段在进入视口时由下淡入。

第 3 段 · 不依赖 IntersectionObserver，纯 CSS 实现。

第 7 段 · 阅读进度也可以放在页面顶部（本页就是）。
第 8 段 · 适用于落地页 / 长文章 / 教程类内容。

14 / Performance Rules

动效性能规则

永远只动 transform / opacity，避免重排

  • 用 transform 和 opacity 实现动效（GPU 加速 / 不触发 layout）
  • 用 will-change: transform 提前告知浏览器（用完移除）
  • 悬停动效不超过 200ms（用户感知阈值）
  • 页面切换控制在 320–600ms（不打断思维流）
  • 同时动效元素不超过 5 个（认知负载）
  • 提供 prefers-reduced-motion 降级（前庭敏感者）
  • 禁用 top / left / width / height 动画 · 触发 layout
  • 禁用 box-shadow 高频动画 · 重绘代价高
  • 禁用 filter 在动画属性上（除非必要）
  • 不要堆叠 3 种以上 同时动效（视觉噪音）
